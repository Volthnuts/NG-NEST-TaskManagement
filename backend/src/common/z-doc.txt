1. Decorators (ป้ายกำกับ / ทางลัด)
 get-user
    เอาไว้ทำ: ทางลัดดึงข้อมูลคนที่ล็อกอินอยู่ (User) ออกมาใช้งาน
    ใช้ตรงไหน: ใช้ใน Controller ตรงวงเล็บของฟังก์ชัน (Param)
    ทำไมต้องใช้: ปกติข้อมูล User จะฝังอยู่ใน req.user ซึ่งยาวและพิมพ์ยาก ถ้าใช้ Decorator นี้ จะเหลือแค่ @GetUser() สั้นๆ
    ใช้อย่างไร: @Get() -> findAll(@GetUser() user: any) { ... }
 roles
    เอาไว้ทำ: ติดป้ายบอกว่า API เส้นนี้ "ใครเข้าได้บ้าง"
    ใช้ตรงไหน: แปะไว้ บนหัวฟังก์ชัน ใน Controller
    ทำไมต้องใช้: เพื่อกำหนดสิทธิ์ เช่น บางเส้นให้เฉพาะ OWNER เข้า บางเส้นให้ MEMBER เข้าด้วย
    ใช้อย่างไร: @Roles('OWNER') แปะไว้เหนือ @Post() หรือ @Get()

2. Guards (ด่านตรวจ / ตำรวจ)
 jwt-auth
    เอาไว้ทำ: ตรวจบัตรประชาชน (Token) ว่าล็อกอินหรือยัง
    ใช้ตรงไหน: แปะบนหัว Controller หรือฟังก์ชัน ด้วย @UseGuards(JwtAuthGuard)
    ทำไมต้องใช้: ป้องกันคนนอกที่ไม่มี Token แอบมายิง API ของเรา
    ใช้อย่างไร: ถ้า Token ปลอมหรือหมดอายุ มันจะดีดกลับ (401 Unauthorized) ทันที
 roles
    เอาไว้ทำ: ตรวจตำแหน่ง (Role) ว่ามีสิทธิ์เข้าห้องนี้ไหม
    ใช้ตรงไหน: แปะคู่กับ JwtAuthGuard บนหัว Controller
    ทำไมต้องใช้: ล็อกอินแล้วไม่ได้แปลว่าทำได้ทุกอย่าง เช่น Member จะมาลบ Project ของ Owner ไม่ได้
    ใช้อย่างไร: มันจะทำงานร่วมกับป้าย @Roles(...) ที่เราแปะไว้ ถ้า Role ไม่ตรง มันจะส่ง (403 Forbidden)

3. Interceptors (เครื่องตกแต่ง / ตัวดักจับ)
 logging
    เอาไว้ทำ: บันทึกเวลาประมวลผล (Log)
    ใช้ตรงไหน: ส่วนใหญ่ตั้งเป็น Global ใน main.ts เพื่อให้จับทุกเส้น
    ทำไมต้องใช้: เอาไว้ดูว่า API เส้นไหนทำงานช้า หรือมีคนยิงมาตอนกี่โมง
    ใช้อย่างไร: มันจะแสดงผลใน Terminal ของเรา เช่น GET /tasks - 50ms
 transform
    เอาไว้ทำ: จับข้อมูลใส่ห่อ JSON ให้เป็นระเบียบ
    ใช้ตรงไหน: ตั้งเป็น Global ใน main.ts
    ทำไมต้องใช้: เพื่อให้หน้าบ้าน (Angular) รับข้อมูลในรูปแบบเดียวกันเสมอ เช่น มีฟิลด์ success, data, message ทุกครั้ง
    ใช้อย่างไร: เราส่งอะไรกลับจาก Service มันจะเอาไปยัดใส่ฟิลด์ "data": [...] ให้เองอัตโนมัติ

4. Interfaces (แม่พิมพ์ / ป้ายบอกชนิด)
 jwt-payload
    เอาไว้ทำ: บอกว่าใน Token 1 ใบ มีข้อมูลอะไรซ่อนอยู่บ้าง
    ใช้ตรงไหน: ใช้ระบุ Type ในไฟล์พวก Guard หรือ Service
    ทำไมต้องใช้: เพื่อให้ตอนเขียนโค้ด พิมพ์ user. แล้วมันขึ้น List มาให้เลือก (userId, email) ไม่ต้องเดา
    ใช้อย่างไร: const payload: JwtPayload = ...
 request-with-user
    เอาไว้ทำ: บอก TypeScript ว่า "Request ของเรามี User แปะมานะ"
    ใช้ตรงไหน: ใช้ใน Guard หรือ Controller เวลาต้องอ้างอิง req.user
    ทำไมต้องใช้: ปกติ Request ของระบบจะไม่มี User ถ้าไม่ใช้ตัวนี้ โค้ดจะแดง (Error)
    ใช้อย่างไร: (req: RequestWithUser) => { ... }

5. Prisma (ท่อส่งข้อมูล)
 prisma.service
    เอาไว้ทำ: ตัวเปิด-ปิดสวิตช์เชื่อมต่อ Database
    ใช้ตรงไหน: "ฉีด" (Inject) เข้าไปใน Constructor ของ Service ทุกตัวที่ต้องใช้ DB
    ทำไมต้องใช้: เป็นศูนย์กลางการคุยกับ Postgres ผ่าน Prisma Client
    ใช้อย่างไร: constructor(private prisma: PrismaService) {} แล้วเรียกใช้ผ่าน this.prisma.task...
